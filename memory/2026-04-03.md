## Architecture Notes
- `_supaClient` = anon key Supabase client (user-facing, RLS applies)
- `_sbMessages` = admin-loaded messages cache (via `admin-data` edge function, service role)
- Edge functions: `stripe-webhook`, `user-messages`, `admin-data`, `notify-email`
- `DB` = local storage fallback database
- Admin uses edge function `admin-data` to load all data server-side (bypasses RLS)
